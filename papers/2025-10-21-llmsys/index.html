0) Executive brief (≤1 page)

What’s solid (2023→2025):

  • Outliers are real and structured. Channel‑wise activation outliers (persistent high‑variance channels) hamper A‑quant; token‑wise “massive activations” are rarer but drive W/A/KV low‑bit failures. Mitigations with migration (SmoothQuant), channel re‑centering/scaling (OS+), and orthogonal rotations (QuaRot/SpinQuant/DFRot) consistently reduce error and enable W4A4 on many LLMs; K/V behave differently and benefit from asymmetric schemes (K per‑channel, V per‑token) at 2‑bit (KIVI). (arXiv)
  • Attention is sparse and skewed. Heads show redundancy (many are prunable/shareable) and sink bias (BOS/initial tokens collect attention mass), which both affect long‑context stability and KV policies (StreamingLLM). (arXiv)
  • Similarity reveals structure. Across layers/heads, Q/K/V projections have low effective rank and high inter‑head similarity in places; CKA/PWCCA reliably capture cross‑layer/head redundancy for clustering/pruning diagnostics. (arXiv)
  • KV cache is the serving bottleneck. Quantization and redundancy‑aware dropping/compression (e.g., H2O heavy‑hitters; LeanKV unified quant+prune) materially cut memory/latency with minor accuracy loss; K and V need different treatment. (NeurIPS Proceedings)
  • Kernel reality matters. Practical wins depend on SDPA/FlashAttention for dense and xFormers BlockSparse (or Triton) for sparse/rotated paths; mis‑matched kernels can negate algorithmic gains. (PyTorch Documentation)

What’s unsettled (and where papers disagree):

  • Rotations: random vs learned. QuaRot shows random (Hadamard/orthogonal) rotations suffice for 4‑bit end‑to‑end on some families; SpinQuant/DFRot find rotation choice is crucial, with learned rotations (Cayley) or refined criteria outperforming random by large margins on hard models (e.g., Llama‑3‑8B). Likely causes: calibration set composition, token‑wise outliers (“massive activations”), and layernorm variants. (NeurIPS Proceedings)
  • Attention sinks: artifact or useful feature? StreamingLLM uses sinks to stabilize long‑context; newer analyses argue sinks are tied to softmax normalization and may be non‑informative “key biases”—replace softmax and sinks diminish. Practical implication: mitigation (prefix/cushion tokens) can help quantization/KV policies but may mask training‑time causes. (arXiv)
  • Head redundancy on modern LLMs. Classical results (BERT/MT) show heavy redundancy; 2024–2025 work shows shareability but also head specialization emerging in later layers/tasks. Need updated audits on Llama‑2/3, Mistral, Qwen‑2, Gemma‑2. (arXiv)

Actionable diagnostics (what to run first on your checkpoints):

  1. Outlier audit: per‑layer/head kurtosis + Hill tail index; token‑ vs channel‑wise split; L∞/L2 per‑channel heatmaps.
  2. Q/K/V spectra: SVD decay + stable rank; mutual coherence; principal angles between col(W_Q), col(W_K).
  3. Attention sparsity: entropy/top‑k mass; sink ratio to BOS/pos‑0; head clustering via CKA/PWCCA.
  4. Quant probes: per‑tensor vs per‑channel vs per‑token INT8/4/2 (with/without rotations/permutations), plus KV‑only quant; correlate ΔMSE→ΔPPL/accuracy. (We provide runnable PyTorch ≥2.x code below.)

3.2 Minimal PyTorch ≥2.x code (sketch)

This is runnable with HuggingFace Transformers and torch ≥2.x; uses SDPA; for BlockSparse, install xFormers. We include only core snippets here.

# Setup
import torch, math, numpy as np
from transformers import AutoModelForCausalLM, AutoTokenizer
torch.set_float32_matmul_precision("high")

model_name = "meta-llama/Llama-2-7b-hf"  # swap: mistralai/Mistral-7B-v0.1, Qwen/Qwen2-7B, ...
tok = AutoTokenizer.from_pretrained(model_name, use_fast=True)
model = AutoModelForCausalLM.from_pretrained(model_name, torch_dtype=torch.bfloat16, device_map="auto", low_cpu_mem_usage=True)
model.eval()

# --- Outlier stats on MLP in/out and attn residuals (per layer/head/channel) ---
import torch.nn.functional as F
def hill_tail_index(x, topm=2048):
    # x: 1D tensor of abs-values
    x = x.flatten().abs()
    x_sorted, _ = torch.sort(x, descending=True)
    xm = x_sorted[:topm]
    # Hill estimator (requires xm>0)
    eps = 1e-12
    return float(1.0 / (torch.mean(torch.log(xm + eps)) - torch.log(xm[-1] + eps)))

@torch.no_grad()
def audit_activations(input_ids):
    hooks, stats = [], dict()
    def make_hook(name):
        def _hook(mod, inp, out):
            x = out.detach().float()
            # per-token channel metrics
            l2 = torch.linalg.vector_norm(x, ord=2, dim=-1)
            linf = x.abs().amax(dim=-1)
            massive = (linf / (x.pow(2).mean(dim=-1).sqrt() + 1e-6) > 6).float().mean().item()
            # channel-wise
            xch = x.reshape(-1, x.shape[-1])
            kurt = torch.mean(torch.mean((xch - xch.mean(0))**4, 0) / (xch.var(0) + 1e-6)**2).item()
            tail = hill_tail_index(xch.abs(), topm=min(4096, xch.numel()))
            stats[name] = dict(massive_frac=float(massive), kurtosis=kurt, hill_tail=tail)
        return _hook

    # register after each RMSNorm before attention/MLP (Llama/Mistral style)
    for i, block in enumerate(model.model.layers):
        hooks.append(block.input_layernorm.register_forward_hook(make_hook(f"layer{i}/pre_attn")))
        hooks.append(block.post_attention_layernorm.register_forward_hook(make_hook(f"layer{i}/pre_mlp")))

    _ = model(torch.tensor([tok.encode("A short calibration passage about LLMs.")] , device=model.device))
    for h in hooks: h.remove()
    return stats

stats = audit_activations(None)
print(list(stats.items())[:2])

# --- Q/K/V spectral & coherence metrics ---
def spectral_metrics_linear(linear):
    W = linear.weight.detach().float().cpu()  # [out, in]
    # SVD (truncated for speed)
    k = min(512, min(W.shape)-1)
    U, S, Vt = torch.linalg.svd(W, full_matrices=False)
    S = S[:k]
    srank = float((S**2).sum() / (S.max()**2 + 1e-9))
    # mutual coherence over columns (approx via random subset)
    cols = F.normalize(W.T, dim=1)
    idx = torch.randperm(cols.shape[0])[:2048]
    C = cols[idx] @ cols[idx].T
    mu = float(C.abs().fill_diagonal_(0).max())
    return {"stable_rank": srank, "mutual_coherence": mu, "sv_ratio": float((S[:10].sum()/S.sum()).item())}

l0 = model.model.layers[0].self_attn
q_metrics = spectral_metrics_linear(l0.q_proj); k_metrics = spectral_metrics_linear(l0.k_proj); v_metrics = spectral_metrics_linear(l0.v_proj)
print(q_metrics, k_metrics, v_metrics)

# --- Principal angles between col-spaces of W_Q and W_K ---
def principal_angles(Wa, Wb, k=128):
    # orthonormal bases via QR
    Qa, _ = torch.linalg.qr(Wa.detach().float().T, mode='reduced')
    Qb, _ = torch.linalg.qr(Wb.detach().float().T, mode='reduced')
    S = torch.linalg.svdvals(Qa.T @ Qb)
    return torch.arccos(torch.clamp(S, 0, 1)).cpu().numpy()  # in radians

angles = principal_angles(l0.q_proj.weight, l0.k_proj.weight)
print("median angle (deg):", float(np.median(angles)*180/np.pi))

# --- Attention sparsity & sink mass (single batch) ---
@torch.no_grad()
def attention_stats(prompt="..."):
    ids = tok(prompt, return_tensors="pt").to(model.device)
    out = model.model(**ids, output_attentions=True)
    ent, topk, sink = [], [], []
    for L, att in enumerate(out.attentions):  # [bs, heads, q, k]
        probs = att[0]  # [H, T, T]
        H,T,_ = probs.shape
        # entropy per head
        e = (-probs * (probs.clamp_min(1e-9)).log()).sum(-1).mean(-1)  # [H]
        ent.append(e.cpu())
        # top-k mass
        topk.append(torch.topk(probs, k=min(8,T), dim=-1).values.sum(-1).mean(-1).cpu())
        # sink ratio (BOS or pos-0)
        sink.append(probs[...,0].mean(-1).cpu())
    return dict(ent=torch.stack(ent), topk=torch.stack(topk), sink=torch.stack(sink))

print(attention_stats("Long input " * 2048)["sink"].mean().item())

4) Synthesis & taxonomy

A. Outlier types → techniques

  • Channel‑wise (persistent high‑var channels): SmoothQuant (migrate), OS+ (shift/scale), AdaDim (per‑IC), AWQ (salient‑channel scaling). Quant target: A & W; Kernel: SDPA unchanged (weight‑side transforms). (arXiv)
  • Token‑wise (“massive”) (rare extreme tokens): DFRot (rotation optimized for tail), PrefixQuant (prefix outliers, static per‑tensor). Targets: W/A/KV; Note: improves static quant significantly. (arXiv)
  • Hidden‑state heavy tails (residual): QuaRot/SpinQuant (global rotations) flatten tails; DuQuant adds permutation to balance blocks. Targets: end‑to‑end W/A/KV at 4‑bit. (NeurIPS Proceedings)

B. KV targets & redundancy

  • Asymmetry (K vs V): KIVI’s analysis supports K per‑channel vs V per‑token (2‑bit) → large mem/throughput wins with small quality loss; LeanKV unifies mixed precision and per‑head sparsity; H2O evicts KVs by attention heavy‑hitters with submodular guarantees. (Proceedings of Machine Learning Research)

C. Head behaviors

  • Redundant/duplicate heads (early layers) ↔︎ good pruning/shareability; induction‑like & sink‑biased heads more in early/mid; later layers more specialized → be cautious with sharing/pruning there. Use CKA/PWCCA to cluster and angle/coherence metrics on (W_Q,W_K). (arXiv)

D. Contradictions & likely causes

  • Rotation methods disagree because of (i) calibration corpus (coverage of rare massive tokens), (ii) normalization (RMSNorm vs LayerNorm shifts tail structure), (iii) bit allocation interaction (CMPQ/AWQ vs pure rotations), (iv) task mix (reasoning vs knowledge). Re‑test on identical settings with our mini‑audit (below).

Appendix: brief derivations & checks

  • Softmax with √(d_k) scaling: Var((q^k/)=1). Without scaling, Var grows with (d_k) → extreme maxima increase → unstable attention entropy.
  • RMSNorm/LayerNorm & quant: per‑token normalization caps tokenwise variance; it does not fix channel‑wise outliers (which are per‑channel global)—hence migration (SmoothQuant) and per‑IC grouping (AdaDim). (arXiv)
  • Why rotations help per‑tensor quant: orthogonal (R) equalizes coordinate energies; for sub‑exponential coordinates, ( E|Rx|) decreases vs (|x|) → smaller clipping ranges for fixed bit budget; borne out empirically by QuaRot/SpinQuant/DFRot. (NeurIPS Proceedings)
